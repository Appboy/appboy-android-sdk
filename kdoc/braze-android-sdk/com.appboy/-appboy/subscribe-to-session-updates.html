open fun subscribeToSessionUpdates(subscriber: IEventSubscriber<SessionStateChangedEvent>)
Content copied to clipboard

Subscribes to Session start and close events. The subscriber callback will be called when a session is created or sealed. The subscriber will be called with a SessionStateChangedEvent. For reference, a session is created or sealed only once. Subscriptions should be cleaned up manually by calling removeSingleSubscription. All events are triggered from background threads and isolate any encountered exceptions. If you need to do work on the UI thread in response to an event, you will need to coordinate that in the IEventSubscriber's trigger method.

Parameters

subscriber

A callback that will be executed when a session is created or sealed.
